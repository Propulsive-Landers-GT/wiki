# Github Access

GitHub is where GTPL stores code, analysis tools, documentation files, and some project repositories. If your subteam uses GitHub, you will need access to the GTPL GitHub organization.

### If You Already Have a GitHub Account

If you already have a GitHub account, DM your subteam lead with:

* Your full name
* Your GitHub username
* Your subteam
* Which repository you need access to if you know it

Your subteam lead will either add you directly or forward your username to someone with organization admin access.

### If You Do Not Have a GitHub Account

If you do not have a GitHub account yet:

1. Go to [https://github.com/](https://github.com/)
2. Create a free account.
3. Pick a username that you are comfortable sharing with the team.
4. Verify your email.
5. DM your subteam lead with your GitHub username.

### After You Are Added

Once you are added to the organization, check your email or GitHub notifications for an invitation. You need to accept the invitation before you can see private repositories.

If you cannot see the repo after accepting the invite, message your subteam lead again and include:

* Your GitHub username
* The repo you are trying to access
* A screenshot of the error or missing repo

### Good GitHub Habits

* Do not commit large data files unless the repo is meant for them.
* Do not upload passwords, API keys, or private credentials.
* Use clear commit messages.
* Ask before changing files you do not understand.
* When in doubt, make a branch, and open a pull request.
