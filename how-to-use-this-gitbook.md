# How to Use This GitBook

## Editing pages

Keep pages short, clear, and easy to scan. Split large topics into child pages.

## Page template

```markdown
# Page Title

## Purpose

## Quick Start

## Details

## Owner

## Last Reviewed
```

## Good documentation habits

* Link to source files instead of duplicating them.
* Put dates on decisions and test results.
* Name the owner for pages that need maintenance.
* Prefer clear procedures over scattered notes.
